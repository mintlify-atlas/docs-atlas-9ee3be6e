## Endpoint

<ParamField path="instanceId" type="string" required>
  The unique identifier for the activity instance. This is a 64-bit integer represented as a string.
</ParamField>

```bash
GET /pgcr/{instanceId}
```

## Description

Get a raw post game carnage report by instanceId. This is essentially the raw data from the Bungie API, with a few redundant fields trimmed off. It should be a subset of the data returned by the Bungie API.

This endpoint is useful if you need to access PGCRs when Bungie's API is down, or if you need the raw Bungie data format instead of RaidHub's normalized format.

<Tip>
  For a more structured response with RaidHub-specific enhancements, use the `/instance/{instanceId}` endpoint instead.
</Tip>

## Response

The response follows Bungie's DestinyPostGameCarnageReportData schema with some fields removed.

<ResponseField name="period" type="string" required>
  ISO 8601 timestamp when the activity occurred
</ResponseField>

<ResponseField name="startingPhaseIndex" type="integer" optional>
  The phase index where the activity started (for activities with checkpoints)
</ResponseField>

<ResponseField name="activityWasStartedFromBeginning" type="boolean" optional>
  Whether the activity was started from the beginning or from a checkpoint
</ResponseField>

<ResponseField name="activityDetails" type="object" required>
  Core activity information from Bungie
  <Expandable title="activityDetails properties">
    <ResponseField name="directorActivityHash" type="integer" required>
      The Bungie.net activity definition hash (uint32)
    </ResponseField>
    <ResponseField name="instanceId" type="string" required>
      The unique instance identifier (64-bit integer as string)
    </ResponseField>
    <ResponseField name="mode" type="integer" required>
      Primary activity mode (DestinyActivityModeType enum)
    </ResponseField>
    <ResponseField name="modes" type="array" required>
      All applicable activity modes for this instance
    </ResponseField>
    <ResponseField name="membershipType" type="integer" required>
      Platform membership type (1=Xbox, 2=PSN, 3=Steam, etc.)
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="activityDifficultyTier" type="integer" optional>
  The difficulty tier of the activity (if applicable)
</ResponseField>

<ResponseField name="selectedSkullHashes" type="array" optional>
  Array of modifier hashes that were active during the activity
</ResponseField>

<ResponseField name="entries" type="array" required>
  Array of player entries with detailed statistics
  <Expandable title="entry properties">
    <ResponseField name="player" type="object" required>
      Player and character information
      <Expandable title="player properties">
        <ResponseField name="destinyUserInfo" type="object" required>
          Destiny user identification
          <Expandable title="destinyUserInfo properties">
            <ResponseField name="iconPath" type="string | null" optional>
              Path to player's icon image
            </ResponseField>
            <ResponseField name="crossSaveOverride" type="integer" required>
              Platform type for cross-save override
            </ResponseField>
            <ResponseField name="applicableMembershipTypes" type="array | null" optional>
              Array of membership types associated with this user
            </ResponseField>
            <ResponseField name="membershipType" type="integer | null" optional>
              Primary membership type
            </ResponseField>
            <ResponseField name="membershipId" type="string" required>
              Destiny membership ID (64-bit integer as string)
            </ResponseField>
            <ResponseField name="displayName" type="string | null" optional>
              Platform-specific display name (legacy)
            </ResponseField>
            <ResponseField name="bungieGlobalDisplayName" type="string | null" optional>
              Bungie global display name
            </ResponseField>
            <ResponseField name="bungieGlobalDisplayNameCode" type="integer | null" optional>
              Bungie global display name code
            </ResponseField>
          </Expandable>
        </ResponseField>
        <ResponseField name="characterClass" type="string | null" optional>
          Character class name (e.g., "Titan", "Hunter", "Warlock")
        </ResponseField>
        <ResponseField name="classHash" type="integer" required>
          Bungie.net class definition hash
        </ResponseField>
        <ResponseField name="raceHash" type="integer" required>
          Bungie.net race definition hash
        </ResponseField>
        <ResponseField name="genderHash" type="integer" required>
          Bungie.net gender definition hash
        </ResponseField>
        <ResponseField name="characterLevel" type="integer" required>
          Character level (50 for level-capped characters)
        </ResponseField>
        <ResponseField name="lightLevel" type="integer" required>
          Character power/light level
        </ResponseField>
        <ResponseField name="emblemHash" type="integer" required>
          Bungie.net emblem definition hash
        </ResponseField>
      </Expandable>
    </ResponseField>
    <ResponseField name="characterId" type="string" required>
      Character ID (64-bit integer as string)
    </ResponseField>
    <ResponseField name="values" type="object" required>
      Dictionary of stat keys to stat value pairs. Common stats include:
      - `assists`: Assists count
      - `kills`: Total kills
      - `deaths`: Total deaths
      - `opponentsDefeated`: Total opponents defeated
      - `completionReason`: How the activity ended
      - `startSeconds`: Seconds from start when player joined
      - `timePlayedSeconds`: Total time played
      - `playerCount`: Number of players in activity
      - `teamScore`: Team score (if applicable)
      - `precisionKills`: Precision/critical kills
      - `weaponKillsGrenade`: Grenade kills
      - `weaponKillsMelee`: Melee kills
      - `weaponKillsSuper`: Super ability kills
      
      Each stat has a `basic` object with `value` (number) and `displayValue` (formatted string).
    </ResponseField>
    <ResponseField name="extended" type="object" optional>
      Extended statistics including weapon details
      <Expandable title="extended properties">
        <ResponseField name="weapons" type="array | null" optional>
          Array of weapons used during the activity
          <Expandable title="weapon properties">
            <ResponseField name="referenceId" type="integer" required>
              Bungie.net weapon definition hash
            </ResponseField>
            <ResponseField name="values" type="object" required>
              Dictionary of weapon-specific stats (kills, precision kills, etc.) in the same format as player values
            </ResponseField>
          </Expandable>
        </ResponseField>
        <ResponseField name="values" type="object" required>
          Additional extended statistics in the same format as entry values
        </ResponseField>
      </Expandable>
    </ResponseField>
  </Expandable>
</ResponseField>

  -H "x-api-key: YOUR_API_KEY"

## Notes

- Response is cached for 24 hours (86400 seconds)
- PGCRs are stored compressed (gzip) in the database and decompressed on request
- BigInt values (instanceId, characterId, membershipId) are converted from strings to BigInt internally, then serialized back to strings in the response
- The `period` field is converted to a JavaScript Date object and serialized as an ISO 8601 string
- Some fields from the original Bungie API response are trimmed to reduce payload size
- This endpoint provides access to historical data even when Bungie's servers are experiencing issues
